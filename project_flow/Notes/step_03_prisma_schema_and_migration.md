## Prisma Version Decision ‚Äî IMPORTANT

**Prisma 7 (initially installed) was downgraded to Prisma 6.**

### Why
Prisma 7 is a breaking major version that removed `url = env("DATABASE_URL")` from `schema.prisma`. It requires a `prisma.config.ts` file and a database adapter package (`@prisma/adapter-pg`). All `project_structure/` reference files were written for Prisma 5/6 syntax. Staying on Prisma 7 would require rewriting every service file that uses PrismaClient.

### Versions in use
```
prisma@6.19.3
@prisma/client@6.19.3
```

### Deprecation warning you will see
```
warn: The configuration property `package.json#prisma` is deprecated and will be removed in Prisma 7.
```
This is safe to ignore ‚Äî it's a v6 warning about v7. It does not affect functionality.

## PrismaClient Singleton (`lib/prisma.ts`)

Reuses a single PrismaClient instance across hot-reloads in development (prevents connection pool exhaustion). Standard pattern for Node.js/Express apps.

In development, all SQL queries are logged to console via `log: ['query', 'error', 'warn']`.

## Key Commands Reference

```powershell
# Run migration after schema change
$env:DATABASE_URL="postgresql://postgres:1234@localhost:5432/learndb"
npx prisma migrate dev --name <migration_name> --schema src/prisma/schema.prisma

# Regenerate client after schema change (without migration)
npx prisma generate --schema src/prisma/schema.prisma

# Open Prisma Studio
npx prisma studio --schema src/prisma/schema.prisma

# Run seed
npx ts-node src/seed.ts
```
