================================================================================
  STEP 01 — MONOREPO SCAFFOLD
  Phase 1 — Foundation
================================================================================

GOAL
────
Create the root directory structure, initialise both projects (API and Web),
install all dependencies, and configure TypeScript and Tailwind so that both
projects compile cleanly before any real code is written.

PREREQUISITES
─────────────
- Node.js 20 installed
- Git repository already initialised (done)
- project_structure/ folder available as reference

PROCESS
───────
1. Create the top-level folder layout at the repository root
   - apps/web/
   - api/
   - shared/types/
   The folders docker-compose.yml and .env.example will be copied in Step 02.

2. Initialise the API project
   - Run npm init inside api/
   - Set entry point to src/server.ts
   - Set name to "learnflow-api"

3. Install all API production dependencies
   - express, cors, cookie-parser         (HTTP server)
   - @prisma/client, prisma               (ORM)
   - bcrypt, jsonwebtoken                 (auth)
   - bullmq, ioredis                      (queue)
   - @anthropic-ai/sdk                    (LLM)
   - elevenlabs                           (TTS)
   - @aws-sdk/client-s3                   (file storage)
   - zod                                  (validation)

4. Install all API development dependencies
   - typescript, ts-node, nodemon
   - @types/express, @types/node, @types/bcrypt,
     @types/jsonwebtoken, @types/cookie-parser

5. Create api/tsconfig.json
   - target: ES2020
   - module: commonjs
   - outDir: dist/
   - rootDir: src/
   - strict: true
   - esModuleInterop: true

6. Add nodemon.json at api/ root for auto-restart during development
   - Watch src/ folder
   - Execute via ts-node

7. Initialise the Web project using Vite with the React + TypeScript template
   inside apps/web/

8. Install all Web production dependencies
   - zustand                              (state management)
   - axios                                (HTTP client)
   - howler                               (audio playback)
   - react-router-dom                     (routing)
   - react-hook-form, zod                 (forms + validation)

9. Install all Web development dependencies
   - @types/howler
   - tailwindcss, postcss, autoprefixer

10. Configure Tailwind CSS
    - Run tailwind init inside apps/web/
    - Add content paths to tailwind.config.js (src/**/*.tsx, src/**/*.ts)
    - Add Tailwind directives to the main CSS file

11. Configure apps/web/tsconfig.json
    - Strict mode on
    - Path alias: @/ → src/ (optional but useful)

12. Copy shared/types/index.ts from project_structure/ into shared/types/

13. Verify both projects compile
    - Run tsc --noEmit inside api/
    - Run tsc --noEmit inside apps/web/ (or npm run build)
    - Both must produce zero TypeScript errors

FILES FROM project_structure/
──────────────────────────────
  shared/types/index.ts              → copy to shared/types/index.ts

DONE WHEN
─────────
  [ ] api/ folder exists with package.json and tsconfig.json
  [ ] apps/web/ folder exists with vite.config.ts and tailwind configured
  [ ] shared/types/index.ts is in place
  [ ] tsc --noEmit passes with no errors in both projects
  [ ] npm run dev works inside apps/web/ (shows a blank Vite page)
